实验题目：利用云平台搭建个人博客

实验学时： 2 实验日期： 2023.3.

实验目的：熟悉个人博客系统的搭建。

具体包括：

参考方案：注册Github账号，搭建Hexo环境并实现个人博客搭建，撰写实验报告。

硬件环境：

联网的计算机一台

软件环境：

Windows or Linux

实验步骤与内容：

  1. 安装环境

  2. 安装node.js和git。在电脑上下载并安装node.js和git的最新版本，确保能够在命令行中使用npm和
    git命令。Node.js：https://nodejs.org/zh-cn
    Git：https://git-scm.com/downloads
    下载 Node.js 和 Git 程序并安装

  3. 连接GitHub
    右键 - > Git Bash Here，设置用户名和邮箱：
    git config –global user.name “GitHub 用户名”
    git config –global user.email “GitHub 邮箱”

  4. 创建密钥

由于之前已经创建过密钥，并完成了设置，跳过此步骤。

  1. 验证连接

打开 Git Bash，输入 ssh -T git@github.com

  1. 创建Github pages仓库
    GitHub 主页右上角加号 - > New repository：

Repository name 中输入 用户名.github.io
勾选 “Initialize this repository with a README”
Description 选填
填好后点击 Create repository 创建。

  1. 安装hexo。在命令行中输入npm install -g hexo-cli来全局安装hexo。

  2. 初始化博客。在命令行中创建一个空文件夹，比如blog，然后进入该文件夹，输入hexo init来初
    始化博客。

  3. 配置博客。在blog文件夹中找到_config.yml文件，用文本编辑器打开，修改其中的一些基本信息，
    比如网站标题、作者、语言、主题等。

  4. 写文章。在blog文件夹中找到source文件夹，在其中创建一个_posts文件夹，然后在_posts文件
    夹中创建一个以.md为后缀的文件，比如hello.md。用文本编辑器打开该文件，在文件开头按照以下格
    式写入文章的元数据：

  5. 生成和预览博客。在命令行中输入hexo generate或者hexo g来生成静态网页文件，在public
    文件夹中可以看到生成的网页。然后输入hexo server或者hexo s来启动本地服务器，在浏览器中输
    入http://localhost:4000来预览博客效果。

  6. 然后在命令行中输入npm install hexo-deployer-git --save来安装部署插件，并输入hexo deploy或者 hexo d来部署博客到github pages上。

结论分析与体会：

hexo是一个基于node.js的静态博客生成器，可以快速地创建和部署个人博客。hexo使用markdown

语言来编写博客文章，markdown是一种简洁易读的标记语言，可以方便地转换为HTML格式。hexo还支
持多种主题和插件，可以根据个人喜好定制博客的外观和功能。本次实验体验了使用hexo搭建个人博客
的过程，并学习了markdown语言的基本语法和用法。通过这种方式，可以方便地管理和发布自己的内容，
并展示自己的风格和特色。
